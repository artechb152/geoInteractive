## scenario item template

**context (2–4 sentences)**
- who/where/goal
- constraints
- the key signal(s) that matter

**task prompt**
- "what should you do next?" / "what is the best response?" / "which diagnosis is most likely?"

**options (if mcq)**
- 4–6 options, all plausible actions
- avoid comedic or obviously wrong choices

**correct answer + rationale**
- rationale must cite the *signal* in the scenario and the rule/procedure

**feedback for each option (recommended)**
- 1–2 lines per option: why this choice helps/harms

**objective mapping + tags**
- objective id
- skill tag (communication, safety, troubleshooting...)

## branching scenario blueprint

for branching content, output a small decision tree like:

| node id | learner sees | learner choice(s) | system response/feedback | score impact | next node |
|---|---|---|---|---:|---|
| n1 | scenario intro... | a) ... b) ... | feedback per choice... | +1 / 0 | n2 / n3 |

rules:
- 3–7 nodes for micro-scenarios; 8–15 for deeper practice.
- keep each node to one decision.
- ensure there is at least one "recovery" path so learners can learn, not just fail.

## sjt scoring options

pick one scoring method and state it clearly:

1. **single best**: 1 point for best, 0 otherwise.
2. **tiered**: best = 2, acceptable = 1, harmful = 0.
3. **rank-order**: learner ranks options; score based on distance from expert rank.

## feedback patterns

use feedback that is:
- **specific**: references a scenario cue
- **actionable**: says what to do instead
- **non-shaming**: focuses on behavior and impact

mini-templates:
- "notice that __. that indicates __. the best next step is __ because __."
- "this choice is tempting because __, but it risks __. a better move is __."
