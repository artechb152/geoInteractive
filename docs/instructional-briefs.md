# Instructional Design Briefs — קורס גיאוגרפיה צבאית

מסמך זה מרכז את תכנון ההדרכה לכל 13 השיעורים, לפי מתודולוגיית `instructional-design-developer`. כל סעיף מהווה את ה"חוזה האינסטרוקציוני" של השיעור — מטרות מדידות, blueprint להערכה, אינטראקציה ייחודית ותכנית הערכה.

## הנחות יסוד לכל הקורס

- **קהל יעד**: 18–23, רובם ללא רקע צבאי קודם בגיאוגרפיה. מצפים לחוויה מודרנית, טמפו מהיר ומשוב מיידי.
- **דרישת ביצועים (Action Mapping)**: בסיום הקורס, הלומד מסוגל **לקרוא מפה צבאית, להעריך מרחב טקטית, ולנתח מוצר GEOINT בסיסי** — לא רק לזכור הגדרות.
- **ברירת מחדל לקושי**: 25% קל / 50% בינוני / 25% קשה.
- **הערכה לפי קירקפטריק**: רמה 1 (סקר חוויה בסוף קורס), רמה 2 (Quiz + תרגול אינטראקטיבי בכל שיעור).
- **שפה ונגישות**: עברית, RTL, פונט Heebo, ניגודיות AAA על רקע כהה, תמיכת מקלדת בכל אינטראקציה, alt-text לכל מפה/תרשים.
- **מבנה קבוע לכל שיעור**: לימוד → תרגול → בדיקת ידע (3 טאבים).

## שיעור 01 — מבוא: מרחב, כוח, אסטרטגיה

| | |
|---|---|
| **מטרת ביצוע** | להסביר איך המרחב משפיע על אסטרטגיה ולסווג החלטות לרמת מלחמה נכונה |
| **רמה קוגניטיבית מובילה** | הבנה + ניתוח |
| **אינטראקציה ייחודית** | `concept-cards` — כרטיסי מושג מתהפכים לארבעת מושגי היסוד (דטרמיניזם, גיאופוליטיקה, מרכז כובד, חיכוך מרחבי) + תרגיל סיווג רמת מלחמה |
| **למה דווקא זה** | מושגים מופשטים — צריך עיגון ויזואלי וחזרה. כרטיסים מנמיכים מחסום וחזרה מרובה משפרת שימור |
| **Blueprint להערכה** | 3 MCQ × ~30 שניות. שאלה 1 = הבחנת מושגים, 2 = סיווג ל"רמת אסטרטגיה", 3 = יישום MDO |
| **טעויות נפוצות לטיפול** | בלבול בין מרכז כובד פיזי למטרה טקטית; חשיבה ש-MDO = "להפעיל את כל הזרועות" במקום "סנכרון ממדים" |

## שיעור 02 — קרטוגרפיה ומפות

| | |
|---|---|
| **מטרת ביצוע** | לתרגם בין נ"צים ולקרוא תבליט נכון ממפה |
| **רמה קוגניטיבית מובילה** | יישום (חישוב) |
| **אינטראקציה ייחודית** | `map-explorer` — מפה טופוגרפית ניתנת ללחיצה: מדידת מרחקים בקנה מידה, המרת קואורדינטות חיה, "מה גובה הנקודה?" |
| **למה דווקא זה** | קרטוגרפיה היא מיומנות "ידיים על המפה" — חייבים תרגול אקטיבי, לא קריאה |
| **Blueprint להערכה** | 3 MCQ + שאלת חישוב יישומית בתרגול. בעתיד: שאלת performance — סמן מסלול במפה |
| **טעויות נפוצות לטיפול** | בלבול קנה מידה גדול/קטן; שכחה לחשב סטיית רשת; הזנת נ"צ בפורמט שגוי (Datum Shift) |

## שיעור 03 — ניווטים ואיכון

| | |
|---|---|
| **מטרת ביצוע** | לבצע איכון עצמאי וניווט מתוכנן ב-GPS-Denied |
| **רמה קוגניטיבית מובילה** | יישום + ניתוח |
| **אינטראקציה ייחודית** | `navigation-sim` — סימולציית חיתוך לאחור: בחר נקודות בולטות, חשב אזימוט חוזר, קבל ציון על דיוק האיכון |
| **למה דווקא זה** | חיתוך לאחור הוא מיומנות שלא נלמדת מצפייה — חייב תרגול אינטראקטיבי |
| **Blueprint להערכה** | 3 MCQ (חישוב אזימוט חוזר, בחירת שיטה, התאמה לתנאי שטח) |
| **טעויות נפוצות לטיפול** | שכחה לקזז סטייה מגנטית; שימוש בניווט עיוור בשטח עם תכסית; חישוב אזימוט חוזר שגוי |

## שיעור 04 — מורפולוגיית שטח

| | |
|---|---|
| **מטרת ביצוע** | לזהות תבניות נוף ולסווג שטח לפי ערך טקטי |
| **רמה קוגניטיבית מובילה** | זיהוי + ניתוח |
| **אינטראקציה ייחודית** | `terrain-classifier` — מקטעי מפה בלייה, סווג: כיפה / אוכף / שלוחה / גיא / שקע. ניקוד + הסבר טעות |
| **למה דווקא זה** | למידת תבניות = חזרה מסיווג + פידבק מיידי. דמוי flashcards אבל מבוסס מפה |
| **Blueprint להערכה** | 3 MCQ (זיהוי, היררכיית שטחים, יישום chokepoint) |
| **טעויות נפוצות לטיפול** | בלבול בין שלוחה לגיא (כיוון V במפה); הנחה ששטח חיוני = שטח שולט (לא תמיד) |

## שיעור 05 — ניידות, תמרון ותכסית

| | |
|---|---|
| **מטרת ביצוע** | לתכנן מסדרון תמרון עם שיקולי עבירות, כיסוי ותכסית |
| **רמה קוגניטיבית מובילה** | סינתזה (תכנון) |
| **אינטראקציה ייחודית** | `mobility-grid` — גריד שטח עם נתוני שיפוע/מסלע/לחות. בחר נתיב לכלי גלגלי או זחלילי, ראה אם עבר ולמה |
| **למה דווקא זה** | תכנון תמרון הוא בעיית אופטימיזציה — חייב משחק בחירות עם השלכות |
| **Blueprint להערכה** | 3 MCQ (סף שיפוע, Cover vs Concealment, פענוח מפנה צומח) |
| **טעויות נפוצות לטיפול** | החלפת מושגי Cover ו-Concealment; התעלמות מאלכסון בשיפוע צד; פענוח שגוי של מפנה דרומי כ"חורש" |

## שיעור 06 — קווי ראייה ו-Viewshed

| | |
|---|---|
| **מטרת ביצוע** | לתכנן עמדות וצירים על בסיס ניתוח ראיה |
| **רמה קוגניטיבית מובילה** | יישום + ניתוח אלגוריתמי |
| **אינטראקציה ייחודית** | `los-simulator` — הצב תצפיתן על מפה ת"ל, ראה Viewshed דינמי. שלב 2: מצא "שטח מת" שמאפשר חדירה |
| **למה דווקא זה** | LOS הוא חזותי במהותו — סימולציה דינמית עוצמתית פי כמה מתרשים סטטי |
| **Blueprint להערכה** | 3 MCQ (גורמי שבירה, פרשנות Viewshed, חשיבות LOS ב-Kill Chain) |
| **טעויות נפוצות לטיפול** | התעלמות מתכסית כשובר LOS; הנחה שגובה אבסולוטי = שליטה (זה תצפית, לא בהכרח שליטה) |

## שיעור 07 — אקלים ועונתיות

| | |
|---|---|
| **מטרת ביצוע** | לשלב מזג אוויר בתכנון מבצעי ובבחירת סנסור |
| **רמה קוגניטיבית מובילה** | ניתוח (תלות ביצועים בתנאים) |
| **אינטראקציה ייחודית** | `weather-impact` — בחר תרחיש (גשם / ערפל / לילה קר). ראה איך משתנים SAR, IR, אופטיקה, טילי לייזר. מד ביצועים חי |
| **למה דווקא זה** | קשר בין אקלים לסנסורים מופשט וטכני — ויזואליזציה הופכת את זה לאינטואיטיבי |
| **Blueprint להערכה** | 3 MCQ (זיהוי מיקרו־אקלים, Thermal Crossover, השלכת תקרת ענן) |
| **טעויות נפוצות לטיפול** | הסתמכות על תחזית אזורית במקום מיקרו־אקלים; שכחה ש-IR נפגע מ-Crossover, לא רק מערפל |

## שיעור 08 — לוגיסטיקה ו-LOC

| | |
|---|---|
| **מטרת ביצוע** | לתכנן ולהגן על קווי אספקה — ולתעדף יעדים תשתיתיים |
| **רמה קוגניטיבית מובילה** | סינתזה + הערכת סיכון |
| **אינטראקציה ייחודית** | `supply-chain` — נהל LOC לאוגדה. בחר MSR/ASR, התמודד עם אירועים (מארב, פיצוץ גשר, הצפה). ניקוד עמידות |
| **למה דווקא זה** | ניהול לוגיסטיקה מבצעית = הימור על תרחישים. משחק בחירות תחת אי־ודאות |
| **Blueprint להערכה** | 3 MCQ (הגדרת ASR, סיכון Push Logistics, ערך מסוף היברידי) |
| **טעויות נפוצות לטיפול** | חשיבה ש-MSR = בהכרח כביש (יכול להיות גם נהר/מסילה); זלזול ב"בטן לוגיסטית" כשמתקדמים מהר |

## שיעור 09 — גיאו־כלכלה ונקודות חנק

| | |
|---|---|
| **מטרת ביצוע** | לזהות תלות גלובלית במשאבים ובנתיבים — והשפעת חסימה |
| **רמה קוגניטיבית מובילה** | ניתוח אסטרטגי |
| **אינטראקציה ייחודית** | `chokepoint-map` — מפת עולם אינטראקטיבית. לחיצה על הורמוז/באב אל־מנדב/מלאקה חושפת תלות אנרגטית, מי משתמש, מה קורה אם נחסם |
| **למה דווקא זה** | היקף גלובלי — חייב מפת עולם דינמית עם מידע "בנגיעה" |
| **Blueprint להערכה** | 3 MCQ (זיהוי באב אל־מנדב, דוקטרינת מהן, הידרו־פוליטיקה) |
| **טעויות נפוצות לטיפול** | חשיבה שרק מדינות יכולות לחסום מיצר (גם ארגון חמוש בנשק רקטי בסיסי) |

## שיעור 10 — לחימה אורבנית (MOUT)

| | |
|---|---|
| **מטרת ביצוע** | לתכנן פעולה במרחב עירוני תלת־ממדי עם אילוצים אזרחיים |
| **רמה קוגניטיבית מובילה** | ניתוח רב־ממדי + שיקול אתי |
| **אינטראקציה ייחודית** | `urban-3d` — מודל עירוני פשוט (גריד מול קסבה). שחק עם זוויות ראיה, איומי גג ומנהרה. סמן אתרים רגישים ופתח ציר הומניטרי |
| **למה דווקא זה** | תלת־ממד עירוני אי אפשר ללמוד ממפה דו־ממדית — חייב מודל אינטראקטיבי |
| **Blueprint להערכה** | 3 MCQ (Enfilade בגריד, אתגר GPS-Denied, הגדרת ציר הומניטרי) |
| **טעויות נפוצות לטיפול** | התעלמות מהממד האנכי; חשיבה שמנהרות = לוגיסטיקה בלבד (יש גם התקפיות) |

## שיעור 11 — גבולות ועומק אסטרטגי

| | |
|---|---|
| **מטרת ביצוע** | להתאים אסטרטגיה למרחב הזמין למדינה |
| **רמה קוגניטיבית מובילה** | ניתוח אסטרטגי |
| **אינטראקציה ייחודית** | `border-strategy` — בחר פרופיל מדינה (עומק רחב / צר / חופי), בחר אסטרטגיה (התרעה / יוזמה / עומק טכנולוגי). ראה תוצאה מול תרחיש פלישה |
| **למה דווקא זה** | מאלץ לנמק בחירה — לא רק לזכור שעומק צר = יוזמה מהירה |
| **Blueprint להערכה** | 3 MCQ (אילוץ עומק צר, חיישנים כתחליף, גבול מלאכותי) |
| **טעויות נפוצות לטיפול** | הנחה שגבול טבעי תמיד "טוב יותר" — לא בהכרח אם הוא חוצה אוכלוסייה אחת |

## שיעור 12 — GEOINT וחישה מרחוק

| | |
|---|---|
| **מטרת ביצוע** | לבחור פלטפורמת איסוף ולהצליב סנסורים לחשיפת הונאה |
| **רמה קוגניטיבית מובילה** | סינתזה (בחירת כלי) + ניתוח (הצלבה) |
| **אינטראקציה ייחודית** | `sensor-fusion` — חמ"ל מיני: קבל מטרה ממקורות שונים (אופטי, SAR, IR, OSINT). הצלב — האם מטרה אמיתית או דמה? |
| **למה דווקא זה** | זו מיומנות הליבה של אנליסט GEOINT — חייב תרגול הצלבה אקטיבי |
| **Blueprint להערכה** | 3 MCQ (יתרון SAR, מהפכת Swarm, הצלבה לחשיפת הונאה) |
| **טעויות נפוצות לטיפול** | חשיבה שאופטיקה היא "האמת" — בלי SAR לא רואים מתחת לעננים ולרשת הסוואה |

## שיעור 13 — GIS מבצעי

| | |
|---|---|
| **מטרת ביצוע** | להפעיל ניתוח GIS לתכנון פעולה ולהערכת איום |
| **רמה קוגניטיבית מובילה** | יישום (כלים) + סינתזה |
| **אינטראקציה ייחודית** | `gis-layers` — בנה Cost Surface ידנית: בחר שכבות (תבליט, איומים, צירים), משקלל, וקבל Least-Cost Path. השווה לפתרון מומלץ |
| **למה דווקא זה** | GIS הוא כלי — לומדים מהפעלה. שכבות בלי לראות אותן בעבודה = קונספט מופשט |
| **Blueprint להערכה** | 3 MCQ (ראסטר vs וקטור, Cost Surface, Network Analysis) |
| **טעויות נפוצות לטיפול** | בחירת ראסטר לאובייקטים בודדים; משקלול לא נכון של איומים מול שיפוע |

## סיכום: מטריצת אינטראקציות

| # | שיעור | אינטראקציה | פיתוח |
|---|---|---|---|
| 01 | מבוא | concept-cards | קל — DOM אנימציה |
| 02 | קרטוגרפיה | map-explorer | בינוני — Leaflet/MapLibre |
| 03 | ניווטים | navigation-sim | בינוני — Canvas + מתמטיקה |
| 04 | מורפולוגיה | terrain-classifier | קל — תמונות + ניקוד |
| 05 | ניידות | mobility-grid | בינוני — גריד SVG |
| 06 | קווי ראייה | los-simulator | קשה — DEM + raycast |
| 07 | אקלים | weather-impact | קל — סליידרים + מד ביצועים |
| 08 | לוגיסטיקה | supply-chain | בינוני — graph + אירועים |
| 09 | גיאו־כלכלה | chokepoint-map | קל — מפת עולם + popovers |
| 10 | אורבני | urban-3d | קשה — three.js |
| 11 | גבולות | border-strategy | קל — בחירות + תוצאה |
| 12 | GEOINT | sensor-fusion | בינוני — תמונות + לוגיקת הצלבה |
| 13 | GIS | gis-layers | בינוני — Canvas שכבות |

המלצה: להתחיל במימוש מלא של **שיעור 02 (map-explorer)** — הוא מייצג חלק נכבד מהאתגר הטכני, ויאפשר לאמת את ה-pipeline לפני העמקה בשאר.
