המרכזית: בסוף הקורס הלומד מסוגל **(1) לקרוא מפה · (2) להעריך מרחב טקטית · (3) לנתח מוצר
GEOINT בסיסי**. **השדרוג הזה משרת את עמוד 3 — "לנתח מוצר GEOINT" — שהוא הכי פחות מתורגל
בפועל היום.** קהל יעד 18–23 ללא רקע צבאי; עברית, RTL; טון רציני-מבצעי אך נגיש.
**אינטראקציה = הדרך ללמוד את המושג, לא קישוט. התוצאה היא החלטה מבצעית, לא ניקוד ריק.**

### סטאק ומבנה
- Next.js + React + TypeScript + Tailwind, `output: 'export'` (סטטי ל-SCORM). ⚠️ להיזהר מ-hydration mismatch — אין `Math.random` ב-render/`useState` initializer.
- כל שיעור = `src/components/lessons/topic-XX/`; `TopicNNLesson` מזין מערך `SCENES` ל-`PagedLearn`; מצב ב-hash (`#scene-<id>`); ניווט בין תתי-נושא דרך אירועי DOM `learn:next`/`learn:prev` (אל תשתמש בשמות אלה לכפתורים פנימיים).

### עוגני מערכת העיצוב
משטחים: `surface`, `surface-elevated`, `bg-elevated`, `bg-accent`. גבולות: `border`, `border-strong`, `border-subtle`. טקסט: `fg`, `fg-muted`, `fg-dim`. אקסנט: `accent`, `accent-hover`, `brand-dark`. סטטוס: `status-ok` (נכון/אמת), `status-warn`, `status-danger` (שגוי/הונאה). עזרים: `chip`, `ease-snap`, `gradient-text`. רכיבים משותפים: `Icon` (`@/components/Icon`, `IconName`; אייקונים שימושיים: `eye`, `satellite`, `crosshair`, `bolt`, `spark`, `check`), `InsightCard` (`@/components/lesson/InsightCard`), `SceneHeader` (מקומי). אנימציה: `framer-motion`, `AnimatePresence`, `useReducedMotion`, `easeSnap`=`cubic-bezier(0.22,1,0.36,1)`. תוויות SVG: `paintOrder="stroke" stroke="#ffffff"`. מקור אמת לצבעים: `tailwind.config.ts` + `docs/palette.md`.

## חלק 3 — מפרט הוראתי

### מטרת ביצוע
בהינתן מטרה חשודה ו-4 מקורות איסוף, הלומד **בוחר אילו מקורות להפעיל, מצליב את הראיות, ומכריע נכון "אמיתי / דמה"** — ומזהה איזה מקור היה מכריע ולמה.

### מטרות למידה
| מזהה | מטרה (מדידה) | רמה קוגניטיבית | ראיה למאסטרי |
|---|---|---|---|
| L1 | להסביר מה כל מקור (אופטי/SAR/IR/OSINT) רואה ומה מגבלתו | הבנה | פתיחת המקורות + קריאת הראיה |
| L2 | להצליב מקורות ולהכריע אמת/הונאה נכון | ניתוח / הערכה | ≥ הכרעה נכונה ברוב התרחישים |
| L3 | לזהות שאופטי לבדו אינו "אמת" — ולזהות את המקור המכריע | הערכה | בחירת המקור המכריע הנכון + מענה לחידון |

### יישור להערכה (קירקפטריק רמה 2)
שאלות ה-MCQ הקיימות של שיעור 12 ב-`quizzes.ts` (יתרון SAR, הצלבה לחשיפת הונאה). לוודא שקיימת שאלה על **הצלבה לחשיפת דמה** ועל **מגבלת האופטי**; התרגיל הוא התרגול לפני אותן שאלות.

### עקרונות שהמימוש חייב לקיים
1. **התוצאה היא הכרעה מבצעית**, לא ניקוד ריק; המשוב מסביר *למה ההצלבה חשפה את ההונאה*.
2. **קונקרטי מנצח מופשט** — כל מקור מחזיר ראיה קונקרטית ("אופטי: נראה טנק" · "IR: אין חתימת חום" → דמה מתנפח).
3. **מורכבות נבנית** — תרחיש ראשון פשוט (הונאה מובהקת), אחרונים מורכבים (למשל מטרה אמיתית שאופטי מפספס מתחת לעננים אך SAR חושף).

## חלק 4 — Storyboard מסך-אחר-מסך

**מיקום:** סצנה חדשה (id `fusion`, שלב "12.x") שנרשמת ב-`Topic12Lesson` **לפני** `RecapScene` — כך היא חותמת את השיעור (ואת הקורס) בשיא. פותחים ב-`SceneHeader` שממסגר: *"אתה אנליסט GEOINT. על השולחן מטרה חשודה. אף מקור בודד לא נותן את התשובה."*

### מסך A — קבלת המשימה (מצב התחלתי)
```
┌─────────────────────────────────────────────────────────┐
│  תרחיש 1/3 · מטרה חשודה בגזרה                            │
│  הדיווח: "זוהה טנק בקו העצים." הפעל מקורות והכרע.        │
│                                                         │
│  מקורות איסוף (לחץ להפעיל):                              │
│  [ 👁 אופטי ]  [ 📡 SAR ]  [ 🔥 IR/תרמי ]  [ 🌐 OSINT ]  │
│                                                         │
│  ┌─ ראיות שנאספו ───────────────────────────────────┐   │
│  │  (ריק — הפעל מקורות כדי לחשוף ראיה)               │   │
│  └──────────────────────────────────────────────────┘   │
│                                                         │
│  ההכרעה שלך:   [ מטרה אמיתית ]   [ דמה / הונאה ]         │
└─────────────────────────────────────────────────────────┘
```
- 4 כפתורי מקור. לחיצה = הפעלה → נחשפת **כרטיס ראיה** לאותו מקור (טקסט קונקרטי + נטייה: תומך-אמת / תומך-הונאה / לא חד-משמעי). הלומד לא חייב להפעיל את כולם — זה חלק מהלמידה (אבל הכרעה בלי מספיק ראיות → משוב שמכוון).
- שני כפתורי הכרעה (אמת / דמה) — נעולים עד שהופעל לפחות מקור אחד.

### מסך B — איסוף ראיות (הצלבה)
```
  👁 אופטי:  "נראה מִתאר טנק ברור בקו העצים."      → נוטה לאמת
  🔥 IR:     "אין חתימת חום. מנוע קר, אין צוות."   → נוטה להונאה  ⚑
  📡 SAR:    "אין החזר מתכתי משמעותי — החומר דק."  → נוטה להונאה  ⚑
  🌐 OSINT:  "דיווחי רשת על 'תנועת שיריון' — לא מאומת." → לא חד-משמעי
```
- כל כרטיס ראיה מגיע עם אייקון סטטוס (`spark` לאזהרה, `check` לתמיכה). הניגוד בין אופטי (אמת) ל-IR+SAR (הונאה) הוא **לב התרגיל** — כאן הלומד רואה שאופטי לבדו מטעה.

### מסך C — רגע ההכרעה (משוב)
הלומד בוחר "אמת" או "דמה". התשובה ננעלת לתרחיש; מופיע משוב:
```
┌─────────────────────────────────────────────────────────┐
│  ✔ נכון — זו דמה (מטרה מתנפחת).                          │
│  איך ידענו? אופטי "ראה טנק", אבל ההצלבה חשפה: IR לא מצא  │
│  חום ו-SAR לא מצא מתכת. שני חיישנים עצמאיים סתרו את       │
│  האופטי → הונאה. **לקח: אופטי לבדו אינו האמת.**          │
│  המקור המכריע: IR + SAR.            [ תרחיש הבא → ]       │
└─────────────────────────────────────────────────────────┘
```
- **נכון:** ירוק (`status-ok`) + הסבר *למה ההצלבה הכריעה* + שם המקור המכריע.
- **שגוי:** אדום (`status-danger`) + הסבר איזו הצלבה פספס. אם הכריע אחרי מקור אחד בלבד — שורה: *"הכרעת מהר מדי — מקור בודד יכול להטעות; הצלב לפני שמכריעים."*
- (אופציונלי, שלב ב') שאלת המשך: "איזה מקור היה מכריע?" לחיזוק L3.

### מסך D — תרחישים 2–3 (מורכבות עולה)
- **תרחיש 2 (היפוך):** אופטי לא רואה כלום (מתחת לעננים / רשת הסוואה), אבל SAR חושף מטרה אמיתית. מלמד את הצד השני: היעדר ראיה אופטית ≠ אין מטרה.
- **תרחיש 3:** מעורב (OSINT + מקור אחד סותר) — דורש שקלול אמיתי.

### מסך E — סיכום
```
   ✔ 3/3 הכרעות נכונות
   כל הכבוד — אתה יודע להצליב מקורות ולחשוף הונאה.
   זכור: אף מקור בודד אינו האמת. אופטי נחסם מעננים והסוואה;
   SAR חודר; IR חושף חום — או את היעדרו.     [ אפס ונסה שוב ]
```
- ניקוד `{correct}/{total}` ב-chip; משפט חיזוק שממסגר את היכולת ("עכשיו אתה יודע ל…") בקו עם דפוס ה-Recap. איפוס זמין. מעבר לסצנה הבאה (Recap) דרך כפתור PagedLearn הרגיל.

### מודל הנתונים (מעתיקים את הרוח מ-`SensorsScene.tsx`)

type SourceId = 'optical' | 'sar' | 'ir' | 'osint';
type Leaning  = 'real' | 'decoy' | 'inconclusive';

type SourceReading = { evidence: string; leaning: Leaning; decisive: boolean };

type Scenario = {
  id: string;
  brief: string;                         // הדיווח הראשוני (מה שאופטי "רואה")
  truth: 'real' | 'decoy';               // האמת

  { id: 'osint',   label: 'OSINT',    english: 'Open-Src', icon: 'crosshair' },
];

const SCENARIOS: Scenario[] = [ /* 3 תרחישים, סדר קבוע, קושי עולה */ ];

### מודל המצב (זרימה מ-`CoverScene.tsx`)
```ts
const [index, setIndex]     = useState(0);                       // תרחיש נוכחי
const [opened, setOpened]   = useState<Set<SourceId>>(new Set()); // מקורות שהופעלו בתרחיש הזה
const [decision, setDecision] = useState<Record<string, 'real'|'decoy'>>({}); // הכרעה per תרחיש

const scen     = SCENARIOS[index];
const answered = decision[scen.id] != null;
const correct  = SCENARIOS.filter(s => decision[s.id] === s.truth).length;

function openSource(id: SourceId) { if (!answered) setOpened(p => new Set(p).add(id)); }
function decide(v: 'real'|'decoy') { if (opened.size > 0 && !answered) setDecision(p => ({ ...p, [scen.id]: v })); }
function next() { setIndex(i => Math.min(i+1, SCENARIOS.length-1)); setOpened(new Set()); } // מאפסים opened לתרחיש הבא
function reset() { setIndex(0); setOpened(new Set()); setDecision({}); }
```
- **סדר קבוע, לא אקראי** — בונה קושי מדורג ומונע hydration mismatch (אין `Math.random` ב-render).
- כרטיסי הראיה נחשפים ב-`AnimatePresence` (כמו המשוב ב-CoverScene), עם `useReducedMotion` שמכבה את ה-slide.

### נגישות (חובה)
- כפתורי המקורות + כפתורי ההכרעה = `<button>` אמיתיים עם `aria-label` עברי ו-`aria-pressed`. Tab + Enter/Space, `focus-visible`.
- אזור הראיות ואזור המשוב = `aria-live="polite"` (קורא-מסך מכריז ראיה חדשה + תוצאת ההכרעה).
- אין הסתמכות על צבע בלבד: אמת/הונאה ונכון/שגוי מסומנים גם באייקון (`check`/`spark`) וגם בטקסט.

### RTL ותנועה
Logical properties בלבד. אם יש ויזואל מפה/גזרה — **לא להפוך (mirror)**. שורות הכפתורים זורמות RTL. מעבר אחד בכל פעם, `easeSnap`, ~0.25–0.4ש; `prefers-reduced-motion` מכובד.

### היקף MVP מול הרחבה
- **MVP:** 3 תרחישים, 4 מקורות, כרטיסי ראיה, הכרעה אמת/דמה, משוב שמסביר את ההצלבה, ניקוד, איפוס. טקסט-מרכזי (בלי ויזואל כבד) — ריאלי ומהיר.
- **הרחבות:** שאלת "מי היה המקור המכריע?"; ויזואל קטן per-מקור (סימולציית "מסך חיישן" — כמו `ThermalViz` ב-SensorsScene); תרחיש רביעי עם OSINT מטעה (fake news) ללימוד ספקנות למקור פתוח.
