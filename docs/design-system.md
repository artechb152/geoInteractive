# Design System — קורס גיאוגרפיה צבאית

## טון ויזואלי
- **Dark-first**: רקע `#0a0e14` עם הצללות עדינות. מתאים לקריאה ממושכת ולהצגת מפות וסנסורים.
- **Bento-grid + topographic backdrop**: רקע גריד דקיק שמרמז על מפה צבאית בלי להיות "צעצועי".
- **טיפוגרפיה**: Heebo לטקסט גוף, Rubik לכותרות. גם וגם תומכים מלא בעברית.
- **קצבים**: anim ק"ב 200ms עם `cubic-bezier(0.22, 1, 0.36, 1)` — תחושת snap מודרנית.

## רכיבים בסיסיים (Tailwind classes)

- `surface` — כרטיס עם רקע, גבול ופינות מעוגלות
- `surface-elevated` — אותו דבר עם הצללה
- `chip` — תווית קטנה (קושי, תג, סטטוס)
- `btn-primary` / `btn-ghost` — כפתורים
- `gradient-text` — כותרת עם gradient על המילים העיקריות
- `topo-bg` — רקע גריד דקיק

## עקרונות אינטראקציה

1. **פידבק מיידי** לכל פעולה (hover, click, submit) — תוך < 200ms.
2. **מצבי שגיאה ידידותיים** עם הסבר ה"למה" ולא רק ה"מה".
3. **Keyboard-first**: כל אינטראקציה זמינה במקלדת בלבד; `focus-visible` ברור.
4. **RTL נכון**: שימוש בלוגי properties (`me-`, `ms-`, `start-`, `end-`) במקום ימין/שמאל מוחלט.
5. **אין anim על `prefers-reduced-motion`**.

## קונבנציות תוכן

- כותרת שיעור: עד 40 תווים; כותרת קצרה: עד 25 תווים.
- מטרת למידה: תמיד פועל פעולה ("לזהות", "לחשב", "להסביר") — לא "להבין" / "לדעת".
- שאלת MCQ: גזע ברור, 4 חלופות, אחת נכונה, רציונל קצר (1–2 שורות).

## מצבים ייחודיים שדורשים תשומת לב

- **מפות / canvas / SVG**: לוודא שמירה על פרופורציות גם ב-RTL — מפה לא מתהפכת.
- **טבלאות נתונים**: יישור מספרים לימין (start in RTL), טקסט לטבלת תוכן יישור התחלה.
- **מקטעי קוד / נ"צ / אזימוט**: שימוש בפונט mono (`font-mono`) — קל יותר לקריאה ולהשוואה.
